The pipeline

Six layers from keyboard to screen. Each one purpose-built to minimize latency.

Input
IOKit HID
Bypasses the NSEvent queue entirely. Raw keyboard events from IOKit's HID subsystem arrive with sub-millisecond overhead, cutting input latency before any terminal processing begins.
Parse
SIMD (Rust)
ANSI escape sequences parsed 16-32 bytes at a time using SIMD intrinsics in Rust. No byte-by-byte state machine — the parser processes entire cache lines in a single instruction.
Buffer
Lock-Free SPSC
PTY data moves from the reader thread to the parser via a single-producer single-consumer ring buffer. No mutexes, no contention, no kernel transitions.
State
Triple Buffer
Terminal state swaps atomically between three buffers. The writer never blocks the reader. No tearing, no partial frames, no render stalls.
Render
Metal GPU
Hardware-accelerated glyph atlas with dirty region tracking. Only changed cells are re-rendered. The GPU handles compositing, ligatures, and subpixel positioning.
Display
IOSurface
Rendered frames go directly from GPU memory to the display via IOSurface, bypassing the macOS window compositor entirely. Zero extra copies.

Why this matters for AI

AI agents generate massive terminal output

When Claude Code reads a 10MB log file, or Codex streams hundreds of lines of code, most terminals choke. Chau7 doesn't flinch — SIMD parsing handles bulk output at cache-line speed, and dirty region tracking means the GPU only redraws what changed.

Token-heavy sessions with Claude Code stay responsive because rendering never blocks the PTY. The SPSC ring buffer ensures the parser keeps reading even while the GPU is mid-frame. No dropped output, no input lag during heavy generation.

Frequently asked questions

Is Chau7 faster than Alacritty?

Chau7 and Alacritty share similar speed philosophies — both use GPU rendering. Chau7 uses Metal (macOS native) while Alacritty uses OpenGL. Chau7 adds IOSurface direct display and IOKit HID input bypass, which can reduce latency further on macOS. Both are significantly faster than non-GPU terminals.

What GPU is required?

Any Mac with Metal support — that's every Mac from 2012 onwards. Chau7 uses Metal for rendering and IOSurface for compositor bypass, both of which are native macOS technologies.

Does performance degrade with many tabs?

No. Inactive tabs are background-suspended — they receive no render cycles and consume minimal resources. Only the visible tab uses GPU rendering. Tab switching is instantaneous because the terminal state is always in memory.
