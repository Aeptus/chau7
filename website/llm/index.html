What is Chau7

Chau7 is an AI-native terminal for macOS. It detects AI agents running in the terminal, optimizes the tokens they consume, tracks their API costs, and gives them programmatic terminal access through 20 local MCP tools.

Chau7 is free, open source, and runs entirely on your Mac. It makes zero network requests to any Chau7 server. All telemetry data stays local.

The name is a French wordplay: chau + 7 (sept) = chaussette = sock.

Core Concepts

How does Chau7 detect which AI agent is running?

Chau7 detects AI agents by monitoring process names in each terminal tab and matching them against known AI CLI signatures. Chau7 identifies Claude Code, Codex, Gemini CLI, ChatGPT, Copilot, Aider, and Cursor automatically.

When Chau7 detects an AI agent, it applies branded tab colors and logos for that agent. Status indicators show whether the agent is active, idle, or completed.

Chau7 also supports custom detection rules so you can add signatures for additional CLIs beyond the built-in list.

What is Context Token Optimization in Chau7?

Context Token Optimization (CTO) is a Chau7 feature that strips terminal noise before AI agents read terminal output. Chau7 removes ANSI escape sequences, progress bars, spinner frames, and repeated whitespace so agents receive only meaningful text.

Chau7's Context Token Optimization reduces token consumption by approximately 40% per session. CTO requires zero configuration and can be toggled per-tab or globally.

What MCP tools does Chau7 expose and how do they work?

Chau7 exposes 20 local MCP (Model Context Protocol) tools through a built-in MCP server that listens on a Unix socket. AI agents connect via the standard MCP protocol and call these tools to control the Chau7 terminal programmatically.

Chau7's MCP tools cover three categories: tab management (create, list, close, execute, read output), telemetry (runs, sessions, tool calls, transcripts), and read-only resources (tab list, session list, config).

The Chau7 MCP server auto-registers in Claude Code, Cursor, Windsurf, and Codex config files on every launch. No manual setup required.

How does Chau7 track AI API costs?

Chau7 tracks tokens, latency, and dollars per API call through a local TLS/WSS proxy. Chau7 intercepts LLM API traffic on your Mac to count tokens and estimate costs without sending any data externally.

Chau7 displays cost data per call, per session, and per month. All cost tracking data stays local on your filesystem.

How does Chau7's GPU rendering architecture work?

Chau7 renders terminal text using Apple Metal for GPU-accelerated glyph compositing. The Chau7 rendering pipeline uses a glyph atlas and dirty-region redraws to minimize GPU work per frame.

Chau7's parsing backend is written in Rust with SIMD-accelerated ANSI escape sequence processing at 16 to 32 byte width. A lock-free single-producer single-consumer ring buffer connects the PTY reader to the Rust parser.

Chau7 uses triple buffering with atomic state swaps to prevent torn frames. The result is smooth, low-latency terminal rendering even at high throughput.

How does Chau7 Remote work on iOS?

Chau7 Remote is an iOS app that controls the Chau7 macOS terminal from an iPhone. Chau7 Remote connects through an end-to-end encrypted relay using Curve25519 key agreement and ChaChaPoly1305 AEAD.

The Chau7 relay runs on Cloudflare Workers and forwards opaque ciphertext it cannot decrypt. Chau7 Remote features include live terminal output, command input, tab switching, approval gates, and Live Activities on the Lock Screen.

How does Chau7's dangerous command guard work?

Chau7 inspects command text at the terminal input layer before the shell executes it. Chau7 matches against destructive patterns (rm -rf, dd, mkfs), detects hidden Unicode characters, and catches suspicious multi-line pastes.

Chau7 pauses flagged commands for user confirmation. The Chau7 command guard is shell-agnostic, so it works regardless of aliases, prompt themes, or shell-specific behavior.

How does Chau7 record terminal sessions?

Chau7 records terminal sessions with millisecond timestamps for every output event. Chau7's recorded sessions can be replayed with a visual timeline scrubber that shows terminal state at any point in time.

Is Chau7 private? Does it send data externally?

Chau7 runs entirely on your Mac. Chau7 makes zero network requests to any Chau7 server. There is no telemetry sent externally, no analytics callbacks, no crash reporting, no update checks.

Chau7 stores token counts, cost data, session recordings, and AI telemetry as local files on your filesystem. There is no account to create because there is no server to talk to.

What MCP tools does the Chau7 terminal provide?

Chau7 provides 26 MCP tools over a local Unix socket. AI agents call these tools via JSON-RPC to manage tabs, query telemetry, and read terminal state. Below is the complete reference for every Chau7 MCP tool.

tab_list
List all tabs across windows with status, git branch, and CTO state.
tab_create
Open a new tab with optional directory and window targeting.
tab_exec
Execute a command in a tab with prompt-state checking.
tab_status
Get process tree, telemetry run, git branch, and CTO state for a tab.
tab_send_input
Send raw input to a tab for interactive prompts (no newline appended).
tab_close
Close a tab with optional force flag for running processes.
tab_output
Get the last N lines of scrollback from a tab (max 5000 lines, 512KB cap).
tab_set_cto
Override Context Token Optimization setting for a specific tab.
run_get
Get a full telemetry run by UUID.
run_list
List runs filtered by session, repo, provider, date range, or tags.
run_tool_calls
Get every tool call an AI agent made during a run.
run_transcript
Get complete conversation turns for a run (for replay).
run_tag
Tag a run for organization and filtering.
run_latest_for_repo
Get the most recent run for a given repository path.
session_list
List sessions with run counts, filterable by repo.
session_current
Get currently active AI sessions across all tabs.
tab://list
Read-only resource: all open tabs with metadata.
session://list
Read-only resource: active AI sessions.
run://latest
Read-only resource: most recent telemetry run.
config://mcp
Read-only resource: current MCP server configuration.

What is Chau7's technical architecture?

Chau7's architecture spans six layers, from the Swift frontend to the encrypted iOS relay. Here is how the Chau7 terminal is built.

Frontend
Chau7's frontend is Swift + AppKit. Native macOS windowing, tab management, menus, and settings.
Parsing
Chau7's parser is written in Rust with SIMD-accelerated ANSI escape parsing (16 to 32 byte width). A lock-free SPSC ring buffer connects the PTY reader to the parser.
Rendering
Chau7 renders with Apple Metal GPU. Glyph atlas, dirty-region redraws, triple buffering with atomic swap.
MCP Server
Chau7's MCP server runs on a local Unix socket using JSON-RPC over stdio. It auto-registers in Claude Code, Cursor, Windsurf, and Codex.
iOS Remote
Chau7 Remote uses a Cloudflare Workers relay with Curve25519 + ChaChaPoly1305 E2E encryption and Live Activities.
Privacy
Chau7 is 100% local. Zero network requests to Chau7 servers. No telemetry, no analytics, no accounts.
