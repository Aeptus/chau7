What is Process Exit Confirmation in Chau7?

Process Exit Confirmation is a safety feature in the Chau7 terminal that warns before closing any tab with a running process. When you press Cmd+W or Cmd+Q, Chau7 inspects the process tree of every active terminal session.

If any foreground or background processes are still running, Chau7 displays a confirmation dialog listing every active process by name and PID. This dialog gives you the chance to cancel the close before anything is killed.

How do I prevent accidentally closing a terminal tab with a running process?

In Chau7, process exit confirmation is enabled by default. There is nothing to configure. When you close a tab or quit Chau7, it inspects the process tree of every active terminal session automatically.

If SSH connections, development servers, compilers, database clients, or any other processes are still running, Chau7 shows a confirmation dialog listing them by name and PID. You can cancel the close, or proceed if you intended to close the tab.

Can I set rules for specific processes in Chau7?

Chau7's settings panel lets you define per-process rules for exit confirmation. You can mark processes like ssh and docker as always-warn, whitelist shells like zsh or bash as safe-to-close, or set specific processes to never warn.

Chau7's per-process rules can be scoped globally or to specific profiles. This scoping means your production SSH connections can require confirmation while local shell tabs close instantly.

Which terminal emulators offer process exit confirmation?

iTerm2 offers a basic close-tab warning when processes are running. Kitty shows a confirmation dialog in similar situations. Neither lists the specific processes by name and PID, and neither supports per-process rules.

Chau7 goes further by listing every running process by name and PID in the confirmation dialog. Chau7 also supports per-process rules (always warn, never warn, or prompt) and checks all tabs when quitting the entire app with Cmd+Q.

Does Chau7's process exit confirmation work when quitting the entire app?

Yes. When you press Cmd+Q, Chau7 checks all open tabs for running processes. Chau7 presents a single dialog listing every active process across all tabs, letting you cancel the quit or proceed.

This app-level check in Chau7 prevents the common mistake of quitting a terminal and losing multiple active sessions at once. Every running process across every tab is visible in one dialog.

Why process exit confirmation matters

Losing a running process to an accidental Cmd+W is one of the most common terminal frustrations. A half-finished database migration, a running dev server, an SSH session to production: gone in a single keystroke.

Chau7 warns before closing any tab with active processes. Chau7's configurable per-process rules let you silence warnings for processes you do not care about while keeping strict confirmation for critical ones like ssh and docker.
