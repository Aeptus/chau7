Command History Search

Chau7 replaces Ctrl+R with a visual fuzzy search through your command history, ranked by frecency. Find commands by what you remember, not exact text.

What is Command History Search in Chau7?

Command History Search is a built-in feature of the Chau7 terminal for macOS that replaces the default Ctrl+R reverse search with a visual fuzzy-search interface. Chau7 shows your entire command history in a filterable, ranked list instead of cycling through matches one at a time.

Chau7's command history search uses frecency ranking to order results. Frecency combines frequency and recency: commands you run often and recently appear at the top, while old one-off commands sink to the bottom.

Is there a better alternative to Ctrl+R?

The default Ctrl+R reverse search in most terminals shows a single line of text, uses exact substring matching, and forces you to cycle through results one at a time. Chau7 replaces this with a full visual search overlay.

Chau7's command history search shows all matching results in a ranked list. Fuzzy matching means characters do not need to be contiguous, so typing "dkr ps" matches "docker compose ps" and "docker ps -a". Frecency ranking ensures the command you want is almost always in the first few results.

How do I fuzzy search through terminal command history?

Press Ctrl+R in Chau7 to open the fuzzy history search. Type any fragments of the command you remember. Chau7 updates results in real time as you type.

Chau7's fuzzy matching algorithm does not require characters to be contiguous. Typing "gco main" matches "git checkout main". Typing "npm tst" matches "npm run test". This lets you find commands by partial memory rather than exact recall.

Select a result with Enter to insert it at the cursor. Press Tab to insert the command for editing before execution.

Does Chau7 work with zsh, bash, and fish history?

Yes. Chau7 detects your active shell and reads from the appropriate history file. Chau7 supports ~/.zsh_history for zsh, ~/.bash_history for bash, and ~/.local/share/fish/fish_history for fish.

Chau7 also supplements shell history with commands executed in the current session. Duplicate entries are collapsed automatically, and failed commands can be optionally filtered out.

Can I delete entries from Chau7 history search?

Yes. In Chau7's command history search, highlight any entry and press the delete shortcut to remove it. Chau7 removes the entry from both the search results and the underlying shell history file.

Deleting history entries in Chau7 is useful for removing commands that contained typos, sensitive information like passwords or tokens, or one-off commands that clutter your search results.

Frequently asked questions

Does Chau7 command history search work with zsh, bash, and fish?

Yes. Chau7 detects your active shell and reads from the appropriate history file: ~/.zsh_history for zsh, ~/.bash_history for bash, or ~/.local/share/fish/fish_history for fish. All standard shell history formats are supported.

Can I delete entries from Chau7 command history search?

Yes. In Chau7's command history search, highlight any entry and press the delete shortcut to remove it from both the search results and the underlying shell history file. This is useful for removing commands that contained typos or sensitive information.

Does frecency ranking replace my shell's history order?

No. Chau7's frecency ranking only affects the display order in Chau7's command history search interface. Your shell's history file remains in chronological order, and native shell features like the up-arrow key continue to work as expected.

What is frecency ranking in Chau7?

Frecency is a ranking algorithm that combines frequency and recency. In Chau7's command history search, commands you run often and recently score higher and appear at the top of results. Old one-off commands score lower and sink to the bottom.

How does Chau7 fuzzy matching differ from exact substring search?

Standard Ctrl+R uses exact substring matching, requiring consecutive characters. Chau7's fuzzy matching allows non-contiguous characters, so "gco main" matches "git checkout main". This lets you find commands by partial memory rather than exact recall.
