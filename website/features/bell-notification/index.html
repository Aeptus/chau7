What is bell notification in Chau7 terminal

Chau7 bell notification is the system that alerts you when a program writes the BEL character (0x07) to your terminal. When a build finishes, a long-running command completes, or a test suite reports results, the program sends a BEL character and Chau7 delivers a configurable notification.

Chau7 provides three independent bell notification modes: visual bell, audio bell, and Dock bounce. Each Chau7 bell mode can be enabled or disabled independently, and you can combine them in any combination per profile.

How to enable visual bell in Chau7 on macOS

Chau7 visual bell briefly flashes the terminal background when a BEL character arrives. To enable Chau7 visual bell, open the Chau7 profile settings for your active profile and toggle the visual bell option on.

Chau7 visual bell is useful when you want a silent notification. For example, you might enable Chau7 visual bell in a monitoring profile that tails logs, so you see the flash without hearing a sound.

How Chau7 audio bell works

Chau7 audio bell plays the macOS system alert sound when a BEL character arrives. The sound Chau7 plays is the same sound configured in macOS System Settings under Sound. To change the Chau7 bell sound, change the macOS system alert sound.

Chau7 audio bell is configured per profile. You can have audio bell enabled in your main development profile but disabled in other Chau7 profiles where sound would be distracting.

How to make Chau7 bounce the Dock icon on bell

Chau7 Dock bounce animates the Chau7 icon in the macOS Dock when a BEL character arrives and Chau7 is not the frontmost application. To enable Chau7 Dock bounce, open the Chau7 profile settings and toggle the Dock bounce option on.

Chau7 Dock bounce only activates when Chau7 is in the background. When Chau7 is the active application, the visual bell and audio bell handle the notification instead. This makes Chau7 Dock bounce ideal for long-running builds or commands where you switch to another application while waiting.

Per-profile bell configuration in Chau7

Chau7 bell behavior is configured per profile. Each Chau7 profile stores its own combination of visual bell, audio bell, and Dock bounce settings independently.

This per-profile control in Chau7 means you can have an audio bell in your main development profile, a silent visual-only bell in a log-monitoring profile, and a Dock-bounce-only bell in a background build profile. Each Chau7 profile's bell settings are completely independent.

Terminal bell not working: Chau7 bell rate limiter

Chau7 includes a bell rate limiter that prevents runaway programs from flooding the system with notifications. If more than 5 BEL characters arrive within one second, Chau7 suppresses further bell notifications until the rate drops.

If your terminal bell appears to stop working in Chau7, the rate limiter may be the cause. A script or program that triggers BEL in a tight loop will hit the Chau7 rate limit. Wait one second and run printf '\a' to confirm the Chau7 bell is still functional.

If running printf '\a' in Chau7 produces no notification at all, check your Chau7 profile settings to confirm that at least one bell mode (visual, audio, or Dock bounce) is enabled.

How Chau7 bell notification compares to other terminals

Chau7 offers three independent, combinable bell modes (visual flash, audio bell, Dock bounce) with per-profile configuration and a built-in rate limiter. Most macOS terminal emulators offer fewer bell options with less flexibility.

macOS Terminal.app supports audio bell and visual bell but does not allow per-profile bell configuration. iTerm2 supports visual bell and audio bell with per-profile settings but does not include a built-in bell rate limiter. Alacritty supports only visual bell with no audio bell or Dock bounce option.

Chau7 is the only macOS terminal emulator that combines all three bell notification modes with per-profile control and automatic rate limiting to prevent notification flooding.

Why Chau7 bell notification matters

The terminal bell is the standard mechanism for programs to request attention. A build finishing, a long-running command completing, a test suite reporting results: all of these use the BEL character. Chau7 bell notification ensures you never miss these events, whether Chau7 is in the foreground or the background.

Frequently asked questions

Can I use a custom sound for the Chau7 terminal bell?

Chau7 uses the system alert sound by default. You can change the system alert sound in macOS System Settings > Sound, which will also change the Chau7 bell sound.

Does the Chau7 Dock bounce work when Chau7 is in the foreground?

No. The Chau7 Dock bounce is specifically for background notification, when Chau7 is not the frontmost application. When Chau7 is in the foreground, the Chau7 visual bell and audio bell provide the notification.

How do I trigger a test bell in Chau7?

Run printf '\a' or echo -e '\007' in your Chau7 terminal. If Chau7 bell notifications are configured correctly, you will see and/or hear the notification immediately.

Why is the terminal bell not working in Chau7?

In Chau7, bell notifications require at least one bell mode (visual, audio, or Dock bounce) to be enabled in your profile settings. If bells stop appearing, Chau7 may be rate-limiting them because more than 5 BEL characters arrived within one second. Run printf '\a' to test. If nothing happens, open Chau7 profile settings and verify that your preferred bell mode is enabled.
