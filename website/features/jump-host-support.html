Jump Host Support

What is Jump Host Support in Chau7?

Jump Host Support is a feature in the Chau7 terminal that lets you configure and connect through multi-hop SSH chains using bastion and jump hosts. Chau7's connection manager handles ProxyJump configuration natively.

When creating or editing an SSH profile in Chau7, you specify one or more jump hosts in the connection chain. Chau7 constructs the correct multi-hop SSH command automatically, handling authentication at each hop including different identity files and usernames for the bastion and the target host.

How do I SSH through a jump host or bastion in Chau7?

In Chau7, open the SSH connection manager and create or edit a profile. Specify one or more jump hosts in the connection chain. Chau7 constructs the correct multi-hop SSH command automatically.

Jump host chains are visualized in Chau7's connection manager as a path diagram showing each hop, its authentication method, and connection status. When connecting, Chau7 displays progress through each hop so you can see exactly where a connection is failing if something goes wrong.

Profiles imported from ~/.ssh/config that use ProxyJump or ProxyCommand directives are parsed and displayed as jump host chains automatically in Chau7. The visual representation makes it easy to understand complex network topologies that would otherwise be buried in config file syntax.

How does Chau7's jump host support compare to other terminals?

Most terminals have no visual jump host management. iTerm2 and Terminal.app require manual ProxyJump configuration in ~/.ssh/config with no visual feedback on the connection chain.

Chau7 is the only terminal that visualizes jump host chains as path diagrams, shows per-hop connection progress, and lets you test connectivity through each hop before opening a session. Chau7 also auto-imports ProxyJump and ProxyCommand directives from your existing SSH config.

How many hops does Chau7 support?

There is no hard limit on the number of hops in Chau7. Chau7 supports arbitrary ProxyJump chains: bastion to bastion to target, or longer if your network requires it.

Each hop in the Chau7 connection chain can have independent authentication settings. This means different identity files, different usernames, and different ports for each bastion in the chain.

Does Chau7 work with ProxyCommand as well as ProxyJump?

Yes. Chau7 supports both ProxyJump and ProxyCommand directives. ProxyJump chains are visualized as hop diagrams in Chau7's connection manager. ProxyCommand entries are displayed with the full command for transparency.

Chau7's connection manager also includes a test button that verifies reachability through each hop. The test reports which stage succeeded or failed without opening a terminal session, making it easy to debug connection issues in complex jump host chains.

Why jump host support matters

Production infrastructure sits behind bastion hosts, requiring multi-hop SSH connections that are tedious to type and easy to misconfigure. Chau7 lets you configure jump host chains visually and connect through them with a single click.

The complexity lives in Chau7's configuration, not in your daily workflow. For teams managing infrastructure behind multiple bastion layers, Chau7's jump host support turns a multi-step manual process into a one-click connection.
