What is JSON-RPC API in Chau7?

The JSON-RPC API is a scripting interface in the Chau7 terminal that exposes terminal automation over a Unix domain socket. Chau7 uses the JSON-RPC 2.0 protocol with standard request/response framing, full error codes, and typed result objects.

Any programming language that can open a socket and send JSON can interact with Chau7's API. No SDK, library, or special tooling is required. A simple socat or netcat command is enough for quick scripting against Chau7.

What protocol does Chau7's API use?

Chau7's API uses JSON-RPC 2.0 over a Unix domain socket. The socket path follows a predictable convention in the user's runtime directory, making it easy to locate from scripts and external tools.

The API surface in Chau7 covers tab management (create, close, list, focus, rename), command execution (send keystrokes, run commands, read output), history access (search, retrieve, export), snippet management (CRUD operations on saved commands), and settings modification (read and write preferences programmatically). Each method is documented with parameter types, return types, and error conditions.

Chau7 as an iTerm2 Python API alternative

iTerm2 offers a Python API for terminal automation, but it requires Python and iTerm2's proprietary library. Chau7's JSON-RPC API is language-agnostic. Any language that can write JSON to a Unix socket can automate Chau7.

Most terminal emulators provide no automation API at all, forcing developers to use AppleScript, osascript hacks, or fragile keybinding simulation. Chau7's JSON-RPC API lets any programming language create tabs, run commands, read output, and manage sessions programmatically. If you can send JSON, you can control Chau7.

Programmatically open terminal tabs on macOS

Chau7 lets you programmatically open terminal tabs on macOS through its JSON-RPC API. Send a tab_create request over the Unix domain socket to open a new tab, optionally specifying the working directory and a command to execute.

Chau7 also supports event notifications via JSON-RPC notification messages. Subscribe to events like tab creation, command completion, directory change, or bell ring to build reactive automation. A single persistent socket connection can issue commands and receive notifications simultaneously, enabling real-time terminal orchestration from external tools.
