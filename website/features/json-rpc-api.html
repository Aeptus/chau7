What is JSON-RPC API in Chau7?

The JSON-RPC API is a scripting interface in the Chau7 terminal that accepts requests over a Unix domain socket at ~/Library/Application Support/Chau7/scripting.sock. Chau7 uses the JSON-RPC 2.0 protocol with standard request/response framing, error codes, and typed result objects.

The API framework is in place and two areas are fully working: history queries via get_history (backed by PersistentHistoryStore with search and limit parameters) and settings via get_settings / set_setting (currently exposing three settings: feature.scriptingAPI, feature.persistentHistory, and history.maxRecords). Tab management and command execution methods are defined in the router but return not_implemented today.

What protocol does Chau7's API use?

Chau7's API uses JSON-RPC 2.0 over a Unix domain socket at ~/Library/Application Support/Chau7/scripting.sock. Any language or tool that can open a Unix socket and send JSON can interact with the API.

Currently working: get_history for searching and retrieving command history with configurable limits, and get_settings / set_setting for reading and writing a small set of preferences. Stubbed but not yet implemented: list_tabs, create_tab, close_tab, get_tab, run_command, get_output, and run_snippet. Event subscriptions and methods like focus_tab or rename_tab do not exist yet.

Chau7 as an iTerm2 Python API alternative

iTerm2 offers a Python API for terminal automation, but it requires Python and iTerm2's proprietary library. Chau7's JSON-RPC API is language-agnostic. Any language that can write JSON to a Unix socket can interact with Chau7.

Most terminal emulators provide no automation API at all, forcing developers to use AppleScript, osascript hacks, or fragile keybinding simulation. Chau7's API currently supports history queries and settings modification. Tab management and command execution are planned but not yet implemented. The API framework and socket are in place, so expanding the method surface is incremental.

What methods are currently working?

The get_history method is fully implemented. It queries PersistentHistoryStore and supports search terms and result limits, returning command history with timestamps and metadata.

The get_settings and set_setting methods work but currently expose only three settings: feature.scriptingAPI, feature.persistentHistory, and history.maxRecords. Tab methods (list_tabs, create_tab, close_tab, get_tab) and command methods (run_command, get_output) are routed but return not_implemented. The list_snippets method returns an empty array and run_snippet returns not_implemented. Event subscriptions are not yet defined.

Most terminal emulators provide no automation API. Developers are forced to use AppleScript, osascript hacks, or fragile keybinding simulation to automate terminal workflows on macOS.

Chau7 is building a proper JSON-RPC 2.0 API over a Unix domain socket. The framework is in place with method routing, error handling, and typed results.
