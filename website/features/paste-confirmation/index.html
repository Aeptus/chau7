Paste Confirmation

Chau7 shows you exactly what you are about to paste before it executes. Multi-line paste gets a safety check.

What is Paste Confirmation in Chau7?

Paste Confirmation is a feature in the Chau7 terminal that shows a confirmation dialog before pasting multi-line content. Chau7 displays the full paste content with syntax highlighting and line numbers so you can review exactly what will be sent to the shell.

Chau7's paste confirmation operates at the terminal emulator level, before bytes reach the shell. This means the protection works regardless of what application is running in the session.

How does Chau7's paste confirmation compare to other terminals?

Most terminals rely solely on bracketed paste mode, which depends on the running application handling paste sequences correctly. Many programs do not handle bracketed paste, leaving users unprotected.

iTerm2 offers a basic paste warning but with no inline editing. Alacritty and Kitty have no paste confirmation at all. Chau7's paste confirmation provides a syntax-highlighted preview with line numbers, inline editing, and protection that works regardless of the running application.

How do I prevent pasting multi-line commands that execute immediately?

Chau7 automatically shows a confirmation dialog when you paste text containing newline characters. Each line is displayed with a line number so you can review exactly what will be sent to the shell.

You can edit the content inline in Chau7's paste dialog, strip trailing newlines, or cancel the paste entirely. This prevents accidental execution of commands from clipboard content, including ClickFix attacks that embed hidden commands in seemingly innocent text.

Can I disable paste confirmation for single-line pastes?

Yes. By default in Chau7, only multi-line pastes trigger confirmation. Single-line pastes go through immediately without a dialog.

You can also configure a character-count threshold in Chau7 so that very long single-line pastes, like base64 blobs or long URLs, are also checked before execution. This gives you fine-grained control over when Chau7's paste confirmation appears.

Does Chau7's paste confirmation replace bracketed paste mode?

No. Chau7's paste confirmation complements bracketed paste mode. Chau7 still sends bracketed paste sequences for applications that support them.

The confirmation dialog in Chau7 provides a second layer of protection that works even when the running application ignores or mishandles bracketed paste sequences. Together, bracketed paste mode and Chau7's confirmation dialog provide two independent layers of paste safety.

Why paste confirmation matters

Pasting from the clipboard is one of the most dangerous terminal actions. A single newline in copied text can execute commands before you see them. ClickFix attacks exploit this by embedding hidden commands in seemingly innocent clipboard content.

Chau7's paste confirmation shows a preview of multi-line pastes and warns about content containing newlines or control characters before anything executes. This protection in Chau7 works at the terminal emulator level, so it cannot be bypassed by the running application.
