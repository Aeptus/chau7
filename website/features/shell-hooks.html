What is Shell Hooks in Chau7?

Shell Hooks is a feature in the Chau7 terminal that lets you run custom logic before and after every command. Chau7 provides three hook types: pre-command hooks, post-command hooks, and directory change hooks.

Chau7 shell hooks fire automatically at key points in the command lifecycle. You configure them once, and Chau7 runs them on every command without you thinking about them.

How do Chau7's pre-command and post-command hooks work?

Chau7 integrates shell hooks that fire at key points in the command lifecycle. A pre-command hook in Chau7 runs after you press Enter but before the command executes. This gives you a chance to log, validate, or modify the environment before execution begins.

A post-command hook in Chau7 runs after the command completes. Chau7 provides the exit code, execution duration, and command text as environment variables. You can use these post-command hooks for reporting, notification, or conditional automation.

What information do Chau7 hooks receive?

Chau7 shell hooks receive structured information as environment variables. These include the command text, working directory, exit status, and timing data.

This structured data from Chau7 lets you build automation like Slack notifications on long-running command completion, automatic git status after cd into a repository, or time tracking for development tasks.

Which shells does Chau7 support for hooks?

Chau7 provides shell integration scripts for Bash, Zsh, and Fish that set up the hook plumbing automatically. The hooks are configured as shell functions or script paths in your shell configuration.

Manual configuration is also possible for other shells by emitting the expected escape sequences that Chau7 listens for.

Can Chau7 trigger actions when you change directories?

Yes. Chau7 directory change hooks fire whenever the working directory changes, whether from an explicit cd command or a directory change triggered by a script.

Chau7's directory change hooks enable workflows like automatically activating Python virtual environments, switching Node.js versions via nvm, or updating the terminal title with project context.

How does Chau7's shell hooks compare to other terminals?

Most terminals rely on manual shell configuration for hooks. iTerm2 offers shell integration but limited hook customization. Alacritty and Kitty have no built-in hook support at all.

Chau7 provides pre-exec, post-exec, and directory change hooks with structured environment variables out of the box. Chau7's shell hooks work with Bash, Zsh, and Fish, and they integrate directly with the terminal's command lifecycle model.

Why Chau7 shell hooks matter

Shell hooks in Chau7 transform the terminal from a passive command executor into an automation platform. Chau7 supports pre-exec, post-exec, and on-error hooks that can trigger notifications, log commands, switch themes, or run arbitrary scripts.

Developers configure Chau7's shell hooks once and they run on every command. No manual intervention. No remembering to trigger a script. Chau7 handles the lifecycle integration automatically.
